## How the pipeline flows

1. User types: `"Show me last week's chicken sales in Coimbatore"`
2. `nl_to_sql()` → Ollama generates SQL using schema + few-shot examples
3. `run_query()` → executes on Oracle, returns rows as dicts
4. `retrieve()` → FAISS lookup for RAG context (only triggered if DB returns empty)
5. `result_to_nl()` → Ollama converts rows + context into: `"Chicken sales in Coimbatore last week totaled ₹8,75,000."`

---

## Decision Logic in pipeline.py

```
user_query
    │
    ├─► nl_to_sql()  ──► SQL
    │
    ├─► run_query(SQL)
    │       ├── rows found  ──► skip RAG
    │       └── rows empty  ──► retrieve() from FAISS
    │
    └─► result_to_nl(query, rows, rag_chunks)  ──► answer
```

The RAG layer acts as a safety net — it only activates when the DB returns nothing, ensuring structured data always takes priority over document retrieval.
